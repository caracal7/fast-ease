Credits

  • fastest-validator - greatest validator we ever used. Not needed for i.js core itself but statically included in full version because its used in 90% of our projects. Maybe will be available as plugin in the future.
  • Walt - great JavaScript-like DSL to wasm compiler. Perfectly fits i.js conception(DSL close to JavaScript) and statically included in full version because it used in a some of our main projects. Maybe will be available as plugin in the future.
  • GPU.js - great JavaScript-like DSL to GLSL compiler. Perfectly fit i.js conception(DSL close to JavaScript) and statically included in experimental i version. Will be released later. Maybe will be available as plugin in the future.
  • nebbia - excellent PoC showing power of template literals
  • lego.js - this great project give us a lot of ideas and motivated to carry out a complete refactoring of i.js(v.0.05). Many parts of code is used. Lego.js makes i.js smaller, simpler, more solid and readable.
  • htmlparser.js - ideas how to easy tokenize HTML
  • proxy-store - simple and usable idea of global state store. Used as internal layer in early versions
  • lazy-events - used in early versions as dependancy to proxy-store
  • s-expression - great example how to parse S-expressions. Used as main core for attributes parsing.
  • parse5 - used in the first i.js PoC as HTML parser. Great for regular HTML parsing but not suitable for custom DSL like i.js is now.
  • lightningDOM - Great and fast as lightning virtual DOM implementation. Used in experimental version of i.js as virtual DOM renderer
